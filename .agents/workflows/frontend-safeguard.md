# UI/UX Preservation & Frontend Safeguard

You are tasked with adding logic, state management, API data integrations, or fixing backend errors within this codebase. The frontend was built using Lovable and contains a highly specific UI design system using React and Tailwind CSS. 

### CRITICAL BOUNDARIES:
1. **Zero Layout Modifications:** Do NOT alter, add, remove, or modify any Tailwind CSS classes within JSX/TSX elements unless explicitly requested. The styling, spacing, padding, margins, colors, and layout must remain 100% untouched.
2. **Preserve DOM Structure:** Do not change the HTML structural hierarchy (nested tags, flexbox setups, grid configurations) of the existing components.
3. **Logic Isolation:** All new React states, custom hooks, API calls, event handlers, and data integration logic must be written *above* the component's `return` statement. Keep functional logic completely decoupled from the layout layout wrappers.
4. **Context-Aware Styling for New Dynamic Elements:** If you must generate a new UI element to reflect logic changes (such as a dynamic error state, a loading skeleton, or an API error message), it must strictly adopt the exact styling primitives (colors, borders, fonts) of the neighboring components. Check the adjacent elements first to ensure seamless integration.
5. **No Style Cleanup:** Even if you notice unusual, redundant, or overly verbose Tailwind utility classes generated by Lovable, do NOT attempt to clean up or optimize them. Treat the existing layout markup as an immutable artifact.
